# Web 工具发布平台设计规格

## 1. 概述

### 1.1 项目定位
一个极简的静态网站门户，作为用户发现和访问各种独立 Web 工具的入口。平台本身不包含工具的运行逻辑，仅作为工具目录和导航。

### 1.2 核心原则
- **零成本运营**：使用免费托管服务，无后端服务器
- **极简功能**：仅提供工具发现入口，无用户系统、评论、评分等复杂功能
- **工具独立**：每个工具完全独立部署，平台仅提供链接

### 1.3 目标用户
- 工具使用者：寻找特定功能的 Web 工具的用户
- 平台维护者：发布和管理工具入口的个人

### 2.2 技术栈

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| 前端框架 | 原生 HTML/CSS/JS | 极简方案，无构建依赖 |

| 数据存储 | 静态 JSON 文件 | 工具列表配置 |

## 3. 页面设计

### 3.1 首页

**布局结构：**
- 顶部：简洁的 Logo + 平台名称 + 简短描述
- 中部：工具分类卡片（网格布局）
- 底部：所有工具列表（卡片式，含搜索框）

**交互功能：**
- 分类筛选：点击分类仅显示该分类工具
- 实时搜索：输入关键词即时过滤工具列表
- 点击工具卡片：新窗口打开工具URL

### 3.2 工具列表卡片

每个工具卡片包含：
- 图标（Emoji 或自定义图标）
- 工具名称
- 简短描述（1行）
- 分类标签
- 可选："精选"标识

## 4. 功能规格

### 4.1 已实现功能

| 功能 | 说明 | 优先级 |
|-----|------|-------|
| 工具列表展示 | 网格布局展示所有工具 | P0 |
| 分类筛选 | 按预设分类过滤工具 | P0 |
| 实时搜索 | 关键词即时过滤 | P0 |
| 响应式设计 | 适配桌面和移动端 | P0 |
| 工具链接跳转 | 点击卡片新窗口打开工具 | P0 |

### 4.2 暂不考虑功能

- 用户注册/登录系统
- 工具评分/评论
- 工具上传/在线编辑
- 下载统计/分析
- 付费/订阅功能

## 5. 部署方案

### 5.1 推荐方案：GitHub Pages

**优势：**
- 完全免费
- 自动部署（push到main分支即自动发布）
- 支持自定义域名
- 全球CDN加速

**部署步骤：**
1. 创建GitHub仓库
2. 上传静态文件（index.html, tools.json, styles.css, app.js）
3. 开启GitHub Pages（Settings > Pages）
4. 可选：配置自定义域名

### 5.2 备选方案：Vercel

适合需要更多功能时（如边缘函数、分析等），但当前需求下GitHub Pages已足够。

## 7. 成功标准

1. 页面加载时间 < 3秒
2. 支持桌面和移动端正常浏览
3. 搜索/筛选响应时间 < 100ms
4. 新增工具仅需修改 tools.json
5. 零后端运维成本
